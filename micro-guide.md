# 🧭 Micro Guide Agent — インターフェース設計

## 音声メイン、でも選べる

---

## 1. 基本原則

**情報はユーザーに最適化され、ガイド UI に示される。**

- 車椅子ユーザー → 段差・傾斜情報が最優先
- 盲導犬ユーザー → 点字ブロック・音声案内が最優先
- ロボット → 幅・衝突リスク・混雑度が最優先
- ベビーカー → 傾斜・床材が最優先
- 高齢者 → 休憩ポイント・距離が最優先

同じ場所でも、見える情報が違う。
同じルートでも、伝え方が違う。

---

## 2. デフォルトは音声

とりあえず、**音声メイン**でいく。

> 「右に曲がって、100 メートル進んでください」
> 「前方 20 メートルに点字ブロックがあります」
> 「次の交差点は T 字路です」
> 「スロープは左側にあります」
> 「混雑しています、迂回ルートを提案します」

目の不自由なユーザーにとって、音声こそが最速のインターフェース。
運転中のロボットにとっても、音声指示は軽量で確実。

**でも、選べる。**

---

## 3. モード選択

ユーザーは自分に合ったモードを選べる。

| モード | 説明 | 対象 |
|--------|------|------|
| 🎧 音声 | 標準。音声ガイド + 簡単なビープ | 盲者・高齢者・ロボット |
| 👁 視覚 | レイヤーメガネ + マップ表示 | 車椅子・ベビーカー・一般 |
| 🖐 触覚 | スマートケーン・振動デバイス | 盲ろう者・聴覚障害者 |
| 📝 テキスト | チャット形式のターンバイターン | 聴覚障害者・ロボットAPI |
| 🌐 複合 | 音声 + 視覚 + 触覚の併用 | 重度重複障害・高齢者 |

---

## 4. ガイド UI の構造

```
[ユーザー種別] → [サーバ] → [最適化された情報] → [ガイド UI] → [ユーザー]
                                                    │
                                                    ├─ 🎧 音声合成
                                                    ├─ 👁 レイヤー表示
                                                    ├─ 🖐 触覚デバイス
                                                    └─ 📝 テキスト出力
```

**ガイド UI は情報を「届け方」ごとにレンダリングするだけ。**

すべての判断はサーバ側で行う。

---

## 5. 音声ガイドの文法

シンプルに、迷わせない。

### 直進
> 「まっすぐ 50 メートル進んでください」

### 右折・左折
> 「次の角を右に曲がってください」
> 「左に曲がって、30 メートル進んでください」

### 注意喚起
> 「前方 10 メートルに段差があります。注意してください」
> 「点字ブロックが途切れています。次のブロックまで 5 メートルです」

### 階層移動
> 「前方 20 メートル、エレベーターがあります」
> 「この階段は車椅子通行不可です。迂回してください」

### 到着
> 「目的地の南口に到着しました」

### エラー・再計算
> 「ルート上に工事を検出しました。迂回ルートを提案します」

---

## 6. ユーザー種別ごとの最適化例

### 盲導犬ユーザー の場合
> 「右に曲がって 50 メートル。点字ブロックが続いています」
> 「交差点です。信号はありませんが、音声案内対応です」

### 車椅子ユーザー の場合
> 「右に曲がって 50 メートル。段差なし、傾斜 3 度です」
> 「前方のエレベーターは幅 90cm、車椅子対応です」

### ロボット の場合
> 「右に曲がって 50 メートル。幅 1.8m、衝突リスク低」
> 「混雑度が高くなっています。迂回を推奨します」

### ベビーカー の場合
> 「右に曲がって 50 メートル。傾斜なし、床材はタイルです」
> 「スロープがあります。押しやすい緩斜面です」

## 7. 実装の優先順位

```
Phase 1: 🎧 音声メイン（MVP）
Phase 2: 👁 レイヤー表示（Web / メガネ）
Phase 3: 🖐 触覚デバイス連携
Phase 4: 🌐 複合モード
```

とりあえず音声で動くものを作る。
そこから広げる。
