# WebXR UI 検討 — レイヤーメガネの現実

## 1. コンセプト

UX.md で描いた「レイヤーが見えるメガネ」を **WebXR AR モード** で実現する。
ブラウザだけで動作、アプリインストール不要。

## 2. 3つのモード

| モード | 表示 | 対象 |
|--------|------|------|
| AR オーバーレイ | カメラ映像に .jsonsg グラフを重ねる | 一般・車椅子・ベビーカー |
| オーディオのみ | 画面オフ、音声だけ | 盲者・高齢者 |
| ハイブリッド | AR + 音声の併用 | 弱視者・ロボット遠隔操作 |

## 3. AR 画面要素

### ノード（地点）
- 改札 = 青い立方体（浮遊）
- エレベータ = 金色の円柱
- 階段 = 赤い三角錐
- 出口 = 緑の球体
- 点字ブロック地点 = 黄色い平面タグ

### エッジ（経路）
- 最適ルート = 虹色の光る線（アニメーション）
- 非選択ルート = 半透明の白線
- 段差あり = 赤い警告線
- 車椅子可 = 青い実線

### HUD 情報
最小限。画面下部に一行だけ表示。

```
▶ 右に曲がって 50m → エレベータ ████████░░ 72%
```

## 4. 操作フロー

```
1. ページを開く → 「AR を開始」ボタン
2. カメラ許可 → QR スキャン待機
3. QR 読み取り → デジタルツイン起動
   └─ ノードが空間に定着
   └─ エッジが可視化
4. 視界を動かすとノードに情報タグ表示
5. 音声ガイド自動開始
6. 目的地到着 → AR セッション終了
```

## 5. WebXR 技術構成

```
TypeScript PWA
├── WebXR AR セッション (immersive-ar)
├── Three.js レンダリング
│   ├── ノード描画 (InstancedMesh)
│   ├── エッジ描画 (Line / TubeGeometry)
│   └── レーベル (CSS2DRenderer)
├── Rust → WASM コアエンジン
│   ├── グラフ読み込み
│   ├── 経路計算
│   └── ノード同定
├── Web Speech API 音声ガイド
└── Service Worker オフラインキャッシュ
```

## 6. パフォーマンス目標

| 指標 | 値 |
|------|-----|
| AR フレームレート | 30fps 以上 |
| WASM バンドル | gzip 50KB 未満 |
| QR → AR 起動 | 3 秒以内 |
| バッテリー消費 | 連続 1 時間動作 |

## 7. 課題

- 動的陰影（屋内照明の変化）
- グラフ位置と実空間のキャリブレーション
- 視覚障がい者には AR は無意味 → オーディオ専用パスを必ず用意
- WebXR の Android Chrome は対応域が限られる

## 8. 優先実装順

```
Phase 1: オーディオのみ（Web Speech API）
Phase 2: 最小 AR（ノード表示のみ）
Phase 3: エッジ表示 + 経路アニメーション
Phase 4: マルチモーダル同期（AR + 音声 + 触覚）
```
